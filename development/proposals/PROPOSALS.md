# Proposals

Design documents and feature specifications for make-bowerbird-test.

## Proposal Lifecycle

### 1. Draft
New proposals start in `draft/`. These are under active development and review.

**Status**: `Draft`

### 2. Accepted
Once a proposal is reviewed, approved, and implemented, it moves to `accepted/`.

**Status**: `Accepted`

### 3. Rejected
If a proposal is not accepted, it moves to `rejected/` with a rationale section explaining why.

**Status**: `Rejected`

## Proposal Format

All proposals must include this header:

````markdown
```
Status:   Draft | Accepted | Rejected
Project:  make-bowerbird-test
Created:  YYYY-MM-DD
Author:   Name or Team
```

---
````
